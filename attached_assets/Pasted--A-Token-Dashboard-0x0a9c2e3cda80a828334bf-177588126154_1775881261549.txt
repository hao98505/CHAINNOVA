### B1. 产出 solidity 合约草案

先放在 `contracts/` 或 `docs/architecture/`，不要部署。

#### 1）`MasterVault.sol`

要求：

* 接收 BNB
* 按 `4286 / 4286 / 1428` bps 分配到 LP / Referral / Marketing
* 记录累计流入、累计分配
* 预留 `owner / multisig` 配置入口
* 加 `ReentrancyGuard`
* 加事件
* 所有比例用 basis points
* 不要设计成资金先到普通钱包

建议至少包含：

* recipient 配置
* `dispatch()` / `distribute()` 逻辑
* `receive()` / `fallback()` 处理
* `totalReceived`
* `totalDistributedToLP`
* `totalDistributedToReferral`
* `totalDistributedToMarketing`

#### 2）`LPRewardVault.sol`

要求：

* `stake / unstake / claim`
* 使用 `accRewardPerShare` 记账
* 奖励资产为 BNB
* 只认指定 LP token
* 提供 `earned(address)` view
* 加事件
* 加重入保护

#### 3）`ReferralVault.sol`

要求：

* `bindReferrer`
* `claim(amount, nonce, deadline, sig)`
* EIP-712 验签
* nonce 防重放
* 记录 `claimed`
* 说明 `claimable` 设计占位
* 不做“后台直接打钱”的思路

#### 4）`MarketingVault.sol`

要求：

* `claimApprovedPayment(amount, nonce, deadline, sig)`
* EIP-712 验签
* nonce 防重放
* 不做任意 `owner withdraw`
* 以“批准后由收款人自己 claim”为设计方向
