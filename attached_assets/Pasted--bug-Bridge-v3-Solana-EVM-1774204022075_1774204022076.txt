====================
第二组：异常用例测试
====================

只做这些异常，不要乱扩展：

1. 非法地址测试
- Solana -> EVM 时输入非法 EVM 地址
- EVM -> Solana 时输入非法 Solana 地址
- 预期：前端或后端拒绝，不能误桥接

2. 余额不足
- Solana 余额不足发起
- BSC / ARB / ETH 余额不足发起
- 预期：前端阻止或链上失败，不能出现错误到账

3. allowance 不足
- EVM 侧不 approve 直接 bridge
- 预期：bridge 失败，不应进入错误成功状态

4. vault 余额不足
- 模拟 Solana Vault SPL 余额不足后执行 EVM -> Solana
- 预期：进入 pendingFailed，不应标记成功

5. ATA 不存在
- 用一个没有 ATA 的 Solana 地址做回桥目标
- 预期：relayer 自动创建 ATA 或明确失败日志

6. watcher / relayer 重启恢复
- 发起一笔测试后中途重启 bridge:watch:solana 或 bridge:watch:evm
- 预期：依赖 state 文件继续处理，不重复、不漏单

7. 重放/重复处理
- 对同一 transferId / 同一 signature 重复触发
- 预期：只处理一次
