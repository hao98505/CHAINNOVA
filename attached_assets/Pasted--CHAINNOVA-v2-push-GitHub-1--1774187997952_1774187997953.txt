- contracts/CNovaWrappedToken.sol

- scripts/configureRoutes.ts

3. 合约需求
CNovaBridge.sol：
- owner 可配置 validator
- owner 可配置 flatFeeWei
- owner 可 configureRoute(localToken, remoteToken, wrapped)
- bridgeOut(localToken, amount, targetChainId, recipientBytes32) payable returns transferId
- completeTransfer(message, validatorSignature)
- processedTransfers 防重放
- 支持两种模式：
  - canonical token: lock/release
  - wrapped token: burn/mint
- 有事件：
  - TokenRouteConfigured
  - BridgeTransferInitiated
  - BridgeTransferCompleted
  - ValidatorUpdated
  - FlatFeeUpdated

CNovaWrappedToken.sol：
- ERC20
- 自定义 decimals
- bridge 地址可设置
- 只有 bridge 可以 mint / burnFromBridge

4. 非常重要：OpenZeppelin 兼容性
不要写出会因为 OZ 版本不兼容而编译失败的代码。
如果用 OpenZeppelin 5：
- Ownable(initialOwner)
- MessageHashUtils + ECDSA 组合
不要用旧版和新版 API 混搭。
编译必须通过。

5. 部署脚本要求
deployBridge.ts：
- 支持按网络部署 bridge / wrapped token
- 使用 env 读取 RPC、PRIVATE_KEY、VALIDATOR、OWNER、FLAT_FEE
- 输出部署地址

configureRoutes.ts：
- 支持配置如下路由：
  - BSC 上 bridge:
    localToken = 0x3e9fc4f2acf5d6f7815cb9f38b2c69576088ffff
    remoteToken = <wrappedForgAI on target>
    wrapped = false
  - opBNB / Arbitrum 上 bridge:
    localToken = <wrappedForgAI on target>
    remoteToken = 0x3e9fc4f2acf5d6f7815cb9f38b2c69576088ffff
    wrapped = true
