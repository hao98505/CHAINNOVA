## 四、铸造代理模态框 (Create Agent Modal)

### 步骤 1：基本信息 (Details)
| 字段 | 描述 | 验证 |
|------|------|------|
| 代理名称 | 文本输入，如 "NEXUS-7" | 必填 |
| 任务描述 | 多行文本，描述代理功能 | 必填 |
| TFLOPS 算力 | 数字输入，默认 100 | 可选 |
| 价格 ($CNOVA) | 数字输入，默认 500 | 可选 |
| 分类选择 | 6 个分类按钮单选 | 默认 Research |

### 步骤 2：元数据 (Metadata)
| 字段 | 描述 | 验证 |
|------|------|------|
| JSON 文件上传 | 点击上传区域模拟文件上传 | 可选 |
| 元数据预览 | 实时显示 JSON 预览（名称、分类、算力、价格、网络） | 自动生成 |

### 步骤 3：确认铸造 (Confirm)
| 字段 | 描述 | 说明 |
|------|------|------|
| 部署摘要 | 汇总显示所有配置信息 | 只读 |
| Gas 预估 | 显示 ~0.000005 SOL | 预估值 |
| 铸造按钮 | 未连接钱包 → 弹出钱包选择；已连接 → 执行铸造 | 模拟交易 |
| 成功界面 | 显示交易签名 + "查看我的代理" 按钮 | ✅ |

| 功能模块 | 描述 | 实现状态 |

## 十一、技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | React 18 + TypeScript |
| 构建工具 | Vite |
| CSS | Tailwind CSS + 自定义赛博朋克类 |
| UI 库 | shadcn/ui |
| 路由 | Wouter |
| 数据管理 | TanStack React Query v5 |
| 动画 | Framer Motion |
| 区块链 | @solana/web3.js + @solana/wallet-adapter |
| 跨链桥 | Wormhole SDK (@wormhole-foundation/sdk) |
| 合约框架 | @coral-xyz/anchor (IDL 占位) |
| 后端 | Express.js (API 路由) |
| 数据库 | PostgreSQL + Drizzle ORM |
